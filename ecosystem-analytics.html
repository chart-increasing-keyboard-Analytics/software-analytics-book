A Literature Survey of Software Analytics

Chapter 7 Ecosystem Analytics

7.1 Motivation

In the modern day and age, the majority of software products make use of external software or libraries to use the functionality of these products, without having to develop this functionality itself. Moreover, multiple languages, such as Python and Rust, provide package managers (pip7 and Cargo8 respectively) which can be used to easily manage this third-party functionality, as well as distribute it.

Together with this comes the growth in open source projects. On platforms such as GitHub9, it is easy and quick to create a new software product, which can be developed, reviewed and used by the whole community.

This in turn leads to intertwined landscapes of software products, which are deemed a software ecosystems. As stated by Messerschmitt and Szyperski (2003), a software ecosystem is ‚Äúa collection of software products that have some given degree of symbiotic relationships.‚Äù Another, similar definition is given by Lungu (2009): ‚ÄúA software ecosystem is a collection of software projects which are developed and co-evolve in the same environment.‚Äù Mens et al. (2013) extends this definition, ‚Äúby explicitly considering the communities involved (e.g.¬†user and developer communities) as being part of the software ecosystem.‚Äù

By performing analysis on these software ecosystems, people aim to generate meaningful insights, which can then be used to improve the efficiency and effectivity of the software development process throughout the lifecycle of the developed software.

In order to perform a survey on the current progress in the field of software ecosystem analytics, we have formulated three research questions:

  • RQ1: What is the current state of the art in software analytics for ecosystem analytics?
  • RQ2: What is the current state of practice in software analytics for ecosystem analytics?
  • RQ3: What are the open challenges in ecosystem analytics, for which future research is required?

Each of these research questions will be answered, using recent papers written in this field of research.

7.2 Research Protocol

In order to select literature to answer the research questions given in the previous section, the survey method suggested by B. Kitchenham (2004a) is used. This method creates a systematic way to select a set of papers, which is relevant to the research question(s).

The search strategy, as described by B. Kitchenham (2004a), are usually iterative and benefit from consultations with experts in the field, amongst other things. Our search strategy can be split in three different types:

  • the initial seed, given by an expert in the field, MSc. Joseph Hejderup
  • a search using a digital search engine, namely Google Scholar10
  • a selection of referenced papers within papers selected before in the above two searches

7.2.2 Digital Search Engine

The second strategy type which is used to select relevant papers for this literature study, is by a digital search engine. In this literature survey, Google Scholar11 is used. The following queries have been used to search for relevant papers:

  • ‚Äúsoftware ecosystems‚Äù AND ‚Äúempirical analysis‚Äù (2018)
  • ‚Äúengineering software ecosystems‚Äù (2014)
  • ‚Äúsoftware ecosystem‚Äù AND ‚Äúempirical‚Äù (2014)
  • ‚Äúsoftware ecosystem analytics‚Äù (2014)
  • ‚Äúsoftware ecosystem‚Äù AND ‚Äúanalysis‚Äù (2017)
  • ‚Äúsoftware ecosystem‚Äù AND ‚Äúempirical‚Äù (2018)

For each of these queries, the results were first filtered by the publish year. These are described by the italic year after each query above. The papers that are filtered are published earlier than the set publish year.

After this filtering, we first determined whether a paper was relevant to the literature survey by examining the title. If it was unclear whether the paper was indeed relevant by only looking at the title, the abstract of the paper was examined closely.
